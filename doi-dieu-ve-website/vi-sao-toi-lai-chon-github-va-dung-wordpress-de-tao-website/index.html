Vì sao tôi lại chọn Github và dùng WordPress để tạo Website

    Chào các bạn! Trong bài viết này thì mình sẽ nêu rõ lí do vì sao mình lại dùng Hosting là Github và CMS là WordPress để tạo nên Website này

    Tại sao lại dùng Github mà không dùng cái khác như WordPress và Blogger?

    Thứ nhất, Github có khả năng tùy biến code của các bạn. Tức là bạn có thể chỉnh sửa được code. Di chuyển được các thư mục và thêm một số tính năng mà bạn hoàn toàn sở hữu. Có thể coi Github là một Hosting cơ bản mà miễn phí tốt nhất!
    Tuy nhiên, nó cũng có một số điều bất lợi như không thực thi được file php, không chạy được CMS động như WordPress, Joomla,…

    Thứ hai, do tính tùy biến tốt như vậy, nên nó có thể làm mới Website đã chỉnh sửa code một cách nhanh chóng mà không khó khăn mất sức.

    Thứ ba, do Blogger và WordPress (bản .com) không có khả năng tùy biến cực tốt như Github. Với lại đa số Blogger và WordPress.com có quá nhiều tin chính trị và định hướng dư luận theo những thành phần phá hoại.

    Thứ tư, Github không sợ bị xóa, không nơm nớp lo sợ như những nơi khác. Bạn toàn quyền sở hữu Website. Về Blogger, Google thì có thể xóa Blog của bạn bất kỳ lúc nào mà không cần báo trước nếu họ cảm thấy nội dung (content) của bạn không phù hợp. Còn về WordPress (.com) thì tương tự như Blogger của Google.
    Bạn có thể kiểm chứng những điều mình nói tại Đây và Đây

    Thế… Nếu Github không hỗ trợ nội dung Website động nhưng tại sao bạn Up WordPress lên nó được??

    Lí do là WordPress được mình đầu tiên up lên XAMPP. Sau khi đã tạo nội dung thì cài một plugin cho phép chuyển mã nguồn WordPress thành Website tĩnh mới có thể sử dụng được.

    Và đôi khi, không phải lúc nào cũng suôn sẻ mà up lên được. Nếu Plugin bị lỗi thì file xuất ra bị lỗi luôn!

    Vậy nếu chứa Website trên đấy có an toàn không? có bị mất dữ liệu không?

    Như mình nói ở trên, Github thì bạn sẽ an tâm hoàn toàn. Nó giống như một Hosting vậy. Tuy nhiên, nó có bất lợi là nếu bạn không quản lý chặt thì rất nguy hiểm. Vì ai cũng có thể chỉnh sửa mã nguồn Website được hết.

    Nhưng nếu bạn dùng tên miền thì có thể an toàn hơn. Vì nếu có tên miền riêng thì khó truy ra nơi lưu trữ.

    Website có thể bị tấn công mã độc hay DDOS hay không

    Website không bị tấn công chèn mã độc vì đây là Website tĩnh. Thậm chí là không thể DDOS vì được bảo kê bởi Github. Tuy nhiên nếu Github bị DDOS quá mạnh và sập thì website của bạn cũng không thoát khỏi đâu. Nhưng đó chỉ là vấn đề về thời gian.

    Github từng có trường hợp bị DDOS lên đến tận 1.35 Tbps mới sập và có lại sau vài giờ. Nên cũng không lo lắng lắm!
